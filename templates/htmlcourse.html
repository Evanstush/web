HTML — Complete Notes & Reference

Deep explanations for HTML tags, attributes, punctuation, and best practices. Read sequentially or jump to the section you need.

Introduction & Document Structure

HTML (HyperText Markup Language) is a markup language used to structure content for the web. It tells browsers what the content is (headings, paragraphs, images, links, etc.) — not how to style it (CSS) or behave (JavaScript).

Minimal valid HTML5 document (explanation follows)

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Page title</title>
  </head>
  <body>
    <!-- page content -->
  </body>
</html>
Line-by-line
  • <!DOCTYPE html> — tells browsers to render using HTML5 rules (standards mode).
  • <html lang="..."> — root element. lang is important for accessibility and SEO (screen readers and search engines).
  • <head> — metadata, CSS links, scripts (deferred), SEO meta tags, and title.
  • <meta charset="utf-8"> — defines character encoding (UTF-8 recommended).
  • <meta name="viewport" ...> — mobile rendering; required for responsive design.
  • <title> — visible in the browser tab and used by search engines.
  • <body> — everything displayed to the user lives here.
Browsers are forgiving, but always write valid HTML to avoid unpredictable behavior, improve accessibility, and help tools (validators, screen readers).

Syntax & Punctuation

Tags / Elements

An element is written as <name>content</name>. Many elements are containers (have opening and closing tags). Some are void elements (no closing tag needed): area, base, br, col, embed, hr, img, input, link, meta, param, source, track, wbr.

Attributes

Attributes are key-value pairs inside the opening tag: <img src=\"a.jpg\" alt=\"desc\" width=\"200\">. Use double quotes around attribute values. Boolean attributes (like disabled) are true when present; no value required.

Character entities

Reserved characters (like <, >, &) must be escaped when used as text: &lt;, &gt;, &amp;. Use named entities for readability (&copy;, &nbsp;) or numeric references.

Comments

<!-- do not put -- inside a comment -->

Case & whitespace

HTML is case-insensitive but lowercase is the convention. Whitespace collapses in normal flow; use <pre> or CSS white-space to preserve it.

HEAD: Meta, Links, Scripts, and Title

The <head> contains metadata about the page. Metadata does not display but is read by browsers, search engines, and social platforms.

Essential meta tags

<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<meta name="description" content="Short summary for SEO">

Open Graph & social tags (example)

<meta property="og:title" content="Page Title">
<meta property="og:description" content="Summary">
<meta property="og:image" content="https://example.com/og-image.jpg">

Title & importance

<title> is required for accessibility and SEO — keep it concise and descriptive (50–70 chars recommended).

link, base, favicon

<link> — CSS, icons, preconnect

<link rel="stylesheet" href="/styles.css">
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="icon" href="/favicon.ico">

base — <base href=\"/sub/\"> sets the base URL for relative links (use carefully — it affects all relative URLs).

style & script

<style>

Use for small page-specific CSS. Prefer external files for caching and maintainability.

<script>

Scripts can block rendering — prefer defer for scripts that run after parsing, or async for independent scripts. Place critical inline scripts carefully.

<script src="/app.js" defer></script>

Document Structure Elements

These semantic structural elements improve meaning, accessibility, and SEO.

<header>

Introductory content for a page or section (logo, site heading, nav). Not the same as <head>.

<nav>

Navigation links. Use landmarks and ARIA labels for complex navigation.

<main>

Main content of the document — should be one per page.

<section>

Thematic grouping — use when content shares a common purpose and usually with a heading.

<article>

Self-contained content (blog post, news article) which could be syndicated.

<aside>

Sidebar or tangential content related to surrounding content.

<footer>

Author info, copyright, links, and closure of a section or page.

Headings: <h1> — <h6>

Use headings to build a document outline. Screen readers and search engines use them to understand structure.

  1. <h1> — page title (one per page preferred).
  2. <h2>—<h6> — subsections. Keep nesting logical (don't skip levels).
Use CSS for visual sizing. Don't use headings solely for styling.

Paragraphs, Line Breaks & Horizontal Rules

<p> for paragraphs. Use <br> for forced line breaks (sparingly), and <hr> for thematic breaks.

Text-level Semantics & Inline Elements

Emphasis & importance

  • <strong> — strong importance (semantic, often bold)
  • <em> — emphasis (semantic, often italic)
  • <b>, <i> — presentational; prefer semantic tags

Small & highlight

  • <small> — fine print
  • <mark> — highlight a run of text
  • <abbr title=\"...\"> — abbreviation with expansion via title

Quotations & citations

  • <q> — short inline quotation (browser may add quotes)
  • <blockquote> — longer block quote; include a cite if available
  • <cite> — reference to a creative work or author

Code & technical text

  • <code> — inline code
  • <pre> — preformatted block (preserves whitespace)
  • <kbd>, <samp>, <var> — keyboard input, sample output, variables

Code Examples & Preformatted Text

Use <pre><code> for code blocks. Include language class for syntax highlighting (if you add a highlighter later): <pre><code class=\"language-html\">.

<pre><code><div>Hello</div></code></pre>

Anchor: <a> — Links

Anchors create links to other pages, anchors (in-page), email, telephone, and downloads.

Common href types

  • https://example.com — external URL
  • /path/page.html — site-relative
  • #section-id — in-page anchor
  • mailto:you@example.com — email
  • tel:+1234567890 — phone link

Security

When using target="_blank" add rel="noopener noreferrer" to prevent the opened page from accessing the opener and to avoid performance/security issues.

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Visit</a>

Images: <img>, <picture>, srcset

Always include alt. For responsive images, use <picture> or srcset with sizes.

Example (srcset)

<img src="small.jpg"
     srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
     sizes="(max-width:600px) 480px, (max-width:1200px) 800px, 1200px"
     alt="Description">

Picture & art direction

<picture>
  <source srcset="wide.jpg" media="(min-width:800px)">
  <img src="default.jpg" alt="...">
</picture>

Use WebP/AVIF for modern browsers to save bandwidth; provide fallbacks for older browsers.

Audio & Video

Use native elements to provide controls and accessibility. Provide captions/transcripts for accessibility and SEO.

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  Your browser does not support audio.
</audio>

<video controls width="640">
  <source src="video.mp4" type="video/mp4">
  <track kind="captions" srclang="en" src="captions.vtt">
  Your browser does not support video.
</video>

Attributes: controls, autoplay (usually muted required), loop, muted, poster.

Embeds: iframe, embed, object

<iframe> is the common way to embed third-party content (YouTube, Google Maps). Use title and loading="lazy" for performance and accessibility. Consider sandboxing if untrusted: sandbox="allow-scripts".

<iframe src="https://www.youtube.com/embed/..." title="Video title" loading="lazy"></iframe>

Lists: ul, ol, li, dl, dt, dd

Use the appropriate list type for the content. For description/definition use <dl>.

<ul>
  <li>Item A</li>
</ul>

<ol>
  <li>Step 1</li;
</ol>

<dl>
  <dt>Term</dt>
  <dd>Definition</dd>
</dl>

Tables: table, thead, tbody, tr, th, td, caption

Use tables for tabular data only. Add <caption> and proper header scopes for accessibility.

<table>
  <caption>Monthly Sales</caption>
  <thead><tr><th scope="col">Month</th><th scope="col">Sales</th></tr></thead>
  <tbody>
    <tr><td>Jan</td><td>100</td></tr>
  </tbody>
</table>

Forms & Inputs — Overview

Forms collect user data. Use proper labels and validation. Never rely solely on client-side validation; always validate server-side too.

Form attributes

  • action — where to send data
  • method — GET or POST
  • enctype — e.g., multipart/form-data for files

Input Types & Attributes

HTML provides many input types that offer built-in UI in browsers.

Common attributes

  • name — key sent to server.
  • id — used for labels (for) and JS selection.
  • placeholder — hint text (not a substitute for label).
  • required — client-side required field.
  • min / max / step / pattern — validation constraints.

Select, Textarea & Button

<select> with <option> for dropdowns. Use <textarea> for multi-line text. Use <button> for actions; prefer type="button|submit|reset".

<label for="country">Country</label>
<select id="country" name="country">
  <option value="ke">Kenya</option>
</select>

<textarea name="message" rows="6"></textarea>

<button type="submit">Send</button>

Semantic HTML & ARIA

Use native semantics first. ARIA provides roles and states to enhance accessibility where native elements fall short.

Examples

<nav aria-label="Main navigation">...</nav>
<button aria-expanded="false" aria-controls="menu">Menu</button>
<div role="dialog" aria-modal="true">Modal content</div>
ARIA should not be used to fix structural problems. Use it to enhance, not replace, semantic HTML.

data-*, template & slot

data-* attributes store custom data: <div data-id="123">. Access via JavaScript: element.dataset.id.

<template>

Holds inert DOM that can be cloned and inserted via JS.

<slot>

Used in Web Components as placeholders for consumer content.

Canvas & SVG (overview)

<canvas> for pixel drawing via JavaScript. <svg> for scalable vector graphics. Both have broad use cases; learn their APIs when needed.

Best Practices, SEO, and Accessibility (Practical)

  1. Semantic HTML: Prefer semantic elements (article, nav, main) over generic divs.
  2. Accessible images: Always provide meaningful alt. If decorative, set alt=\"\".
  3. Forms: Always label inputs. Use fieldset/legend for groups.
  4. Responsive: Use viewport meta, fluid images, and relative units.
  5. Performance: Defer non-critical scripts, use lazy loading (loading=\"lazy\"), compress images, minimize DOM complexity.
  6. SEO: Use descriptive title/meta description, semantic headings, and structured data if needed.
  7. Security: Escape user input, use HTTPS, and avoid inline JS/CSS for CSP simplicity.
