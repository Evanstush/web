Introduction — Where CSS Fits

CSS (Cascading Style Sheets) controls presentation — color, spacing, layout, and visual behavior. HTML structures content; CSS styles it; JavaScript adds behavior. Keep concerns separate for maintainability and accessibility.

Pro tip: start with semantic HTML. Use CSS to enhance, not to fix missing semantics.
Design tokens: centralize colors, spacing, radii using CSS variables for consistency across the site.

Syntax, Selectors & Specificity

Basic Syntax
/* selector { property: value; } */ .btn { color: #fff; background: var(--brand); padding: 0.6rem 1rem; }
Selectors (what to use & when)
SelectorUse
elementGlobal rules for tags (e.g., p, h1)
.classPreferred for components & reusability
#idUnique items; avoid for general styling (high specificity)
[attr]Style by attributes (useful for inputs & states)
:hover / :focusUser interaction states (keyboard + mouse)
:nth-child()Structural patterns without extra markup
Specificity rules (short)

Specificity ranking: inline styles > IDs (#) > classes/attributes/pseudo-classes > elements/pseudo-elements. Use low-specificity class-based rules for maintainability. Avoid !important except as last resort.

Box Model & Display

Box Model (content → padding → border → margin)
/* border-box makes width include padding & border */ *, *::before, *::after { box-sizing: border-box; }

Using border-box simplifies sizing; recommended for most projects.

display: block / inline / inline-block / none

Block elements take full width. Inline elements flow within text. Inline-block allows dimensions. display: none removes from flow (not the same as visibility hidden).

Overflow & scroll containers

Use overflow: auto for scrollable areas; prefer GPU-accelerated transforms for interactive content inside scroll regions.

Colors & Palettes

Color formats
/* hex, rgb(a), hsl(a) */ .selector { color: #0d6efd; } .selector { background: rgb(13,110,253); } .selector { border-color: hsl(220,90%,56%); }

HSL is great for programmatic color changes (lighten/darken by adjusting L).

Create a palette (example)

Keep 3–5 colors and a set of neutrals. Use contrast checkers for accessibility.

Using alpha/transparency

Use rgba() or hsla() for overlays; avoid semi-opaque text over images when readability suffers.

Typography & Text

Font stack & performance
/* prefer system fonts for performance */ body { font-family: Inter, system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif; }

Use font-display: swap when self-hosting or using web fonts to avoid invisible text.

Sizes, line-height & rhythm

Use a scale (e.g., 0.875rem, 1rem, 1.125rem...) and consistent line-height (1.4–1.8) to improve readability. Use rem for scalable sizing and accessibility.

Text properties
p { color: #333; line-height: 1.6; }

Backgrounds, Borders & Shadows

Background images & gradients
/* cover background with center */ .hero { background: url('/hero.jpg') center/cover no-repeat; } .panel { background: linear-gradient(180deg,#fff,#f1f7ff); }
Border radius & shadow
.card { border-radius: 12px; box-shadow: 0 8px 24px rgba(12,40,80,0.06); }

Use consistent radii (e.g., 4,8,12,16) across components for coherence.

Flexbox — One-dimensional Layout

Core concepts & properties
.row { display: flex; gap: 1rem; align-items: center; }

Flexbox excels at aligning items and distributing space in a single axis.

Flex item sizing
.col { flex: 1 1 0; }

Shorthand flex: grow shrink basis. Use carefully to avoid layout jumps.

Practical examples
/* center a loader */ .container { display: flex; align-items: center; justify-content: center; }

CSS Grid — Two-dimensional Layout

Grid basics
.grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; }

Grid controls both rows and columns; ideal for complex layouts and dashboards.

Responsive grids
.cards { display: grid; grid-template-columns: repeat(auto-fit, minmax(220px, 1fr)); gap: 1rem; }
Named areas & alignment
.layout { grid-template-areas: "header header" "sidebar main" "footer footer"; }

Positioning & Sticky

Position property

Use relative for local offsets, absolute off the flow (positioned to nearest ancestor with position), fixed for viewport pinned elements, sticky for hybrid behaviors (excellent for headers/TOC).

Sticky example
aside { position: sticky; top: 1rem; }

Responsive Design & Units

Mobile-first media queries
@media (min-width: 768px) { .container { max-width: 720px; } }

Write base (mobile) styles first, then add breakpoints upward to reduce overrides and improve mobile performance.

Units: rem, em, %, vw/vh

Use rem for consistent scale, % for fluid widths, and viewport units for full-screen artboards. Use clamp() for fluid typography.

Fluid Typography & clamp()

Example
h1 { font-size: clamp(1.5rem, 2.5vw, 2.5rem); }

Clamp provides min, preferred, and max — great for responsive text without many breakpoints.

Transforms & Transitions

Transitions (use for hover/focus)
.btn { transition: transform .18s ease, box-shadow .18s ease; } .btn:hover { transform: translateY(-3px); }

Animate properties that avoid layout (transform, opacity) for performance.

Transforms
.tile { transform: rotate(-2deg) scale(.98); }

Animations & Keyframes

Keyframes & usage
@keyframes fadeUp { 0% { opacity: 0; transform: translateY(8px); } 100% { opacity: 1; transform: translateY(0); } } .card { animation: fadeUp 420ms ease both; }

Respect prefers-reduced-motion for accessibility.

Reduce motion support
@media (prefers-reduced-motion: reduce) { .card { animation: none; transition: none; } }

Custom Properties (CSS Variables)

Define and use variables
:root { --brand: #0d6efd; --gap: 1rem; --text: #1f2937; } .btn { background: var(--brand); padding: var(--gap); }

Variables cascade and can be changed in media queries for theming (dark mode).

Runtime updates & theming

Update variables with JavaScript to switch themes without repainting many rules.

calc(), min(), max(), clamp()

Examples
.container { width: calc(100% - 2rem); } .title { font-size: clamp(1.05rem, 2vw, 1.5rem); }

Use these functions for fluid layouts and to combine different units safely.

Accessibility & Motion

Color contrast & focus

Ensure text meets WCAG contrast. Provide visible focus styles for keyboard users (outline or custom focus ring).

a:focus, button:focus { outline: 3px solid #ffd54f; outline-offset: 3px; }
Reduce motion
@media (prefers-reduced-motion: reduce) { * { animation: none !important; transition: none !important; } }

Performance & Rendering

Rendering tips
  • Avoid expensive layout-triggering animations (width/height/margin). Prefer transform and opacity.
  • Minimize CSS complexity and selector depth for faster matching.
  • Use critical CSS inline and lazy-load non-critical styles.
